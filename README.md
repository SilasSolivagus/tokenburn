# 🔥 tokenburn

**htop for your AI spending.** See where your money goes when using Claude Code, Cursor, aider, and other AI coding tools.

```
$ tokenburn scan --last 7d

🔥 7-day total: $89.40

  Waste detection (4 issues found):

  🔴 23 duplicate requests — $14.20 wasted
     Same file read 11 times in a loop
     → Check if your Agent is re-reading files unnecessarily

  🟡 47 short responses used opus — $18.60 savable
     Responses under 200 tokens generated by expensive models
     → Switch to haiku for these → save $12.50/week

  🟡 Cache hit rate: 6.2%
     Most input tokens are not cached
     → Keep system prompts stable to improve caching

  🟢 3 truncated responses — $2.10 wasted
     Requests hit max_tokens limit

  💰 Potential savings: $28.80/week (32%)
```

## Quick Start

```bash
# Install
npm install -g tokenburn

# Start the proxy
tokenburn proxy start -d

# Configure your tools (one-time)
eval $(tokenburn proxy env)

# See your spending
tokenburn report
tokenburn scan
tokenburn live
```

## Zero Config

tokenburn reads Claude Code's local logs automatically. No proxy needed for basic usage:

```bash
# Just run it — auto-imports logs and shows today's report
tokenburn

# Import with time filter
tokenburn import --since 7d
```


## How It Works

tokenburn runs a transparent local proxy that intercepts API calls to Anthropic and OpenAI. It records token usage and costs in a local SQLite database, then analyzes patterns to find waste.

```
Your AI tool → localhost:10811 → tokenburn (record) → api.anthropic.com
                                     ↓
                              ~/.tokenburn/tokenburn.db
```

**Zero data leaves your machine.** All analysis is local.

## Commands

| Command | Description |
|---|---|
| `tokenburn proxy start [-d]` | Start proxy (foreground or daemon) |
| `tokenburn proxy stop` | Stop background proxy |
| `tokenburn proxy status` | Check if proxy is running |
| `tokenburn proxy env` | Print env vars for shell |
| `tokenburn report` | Cost report (today by default) |
| `tokenburn report --last 7d` | Report for last 7 days |
| `tokenburn report --by model` | Break down by model |
| `tokenburn report --by source` | Break down by tool |
| `tokenburn scan` | Detect waste patterns |
| `tokenburn live` | Real-time dashboard |
| `tokenburn db export --csv` | Export raw data |

### Waste Detection (30 rules)

tokenburn scans for 30 waste patterns across 4 categories:

- **Deep patterns**: duplicate requests, model overuse, retry storms, large file re-reads...
- **Operation analysis**: read-heavy sessions, excessive bash loops, write-rewrite cycles...
- **Context & config**: context drift, missing CLAUDE.md, deep agent trees...
- **Value analysis**: plan underuse/overuse, cheaper model suggestions

```bash
tokenburn scan --fix              # Interactive fix mode
tokenburn scan --rules            # List all 30 rules
tokenburn scan --disable late-night,idle-chat
```

### Optimization

```bash
tokenburn optimize                # Get optimization plans
tokenburn optimize --simulate --model claude-haiku-4-5
tokenburn optimize --apply        # Auto-apply fixes to CLAUDE.md
```
